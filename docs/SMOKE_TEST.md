# SMOKE TEST (ручная проверка)

Ниже — базовый чеклист для ручной проверки после изменений.

## Автоматическая smoke-проверка (headless)

Команда запуска:

```
godot --headless --script res://tests/run_headless.gd
```

Проверяется:
1. DataDB загружается (items/mobs не пустые).
2. SaveSystem сохраняет в тестовую директорию и создаёт backup при перезаписи.
3. LootSystem генерирует лут с валидными item_id и корректными стеками.
4. AppState принимает легальные переходы и блокирует нелегальные.

Логи выводятся в консоль. Код выхода: `0` — все проверки пройдены, `1` — есть хотя бы один FAIL.

1. **Запуск проекта**
   - Шаг: Запустить проект из Godot.
   - Ожидаемый результат: Проект стартует без ошибок, отображается стартовая сцена.

2. **Появление Login UI**
   - Шаг: Дождаться загрузки стартовой сцены.
   - Ожидаемый результат: Отображается экран логина.

3. **Прохождение UI-флоу**
   - Шаг: Ввести логин/пароль и нажать Login.
   - Ожидаемый результат: Переход к экрану выбора/создания персонажа.

4. **Загрузка персонажа / мира**
   - Шаг: Создать персонажа (если нужно), выбрать его и нажать Enter.
   - Ожидаемый результат: Загружается игровая сцена (мир).

5. **Сохранение / загрузка персонажа**
   - Шаг: Открыть меню в мире и выбрать Exit World (или Exit Game) для принудительного сохранения.
   - Шаг: Перезапустить игру, снова выбрать персонажа и войти в мир.
   - Ожидаемый результат: Сохранение и загрузка проходят без ошибок, данные персонажа сохранены.

6. **Базовые интеракции (если есть)**
   - Шаг: Проверить базовые взаимодействия (например, перемещение, выбор цели кликом).
   - Ожидаемый результат: Базовые интеракции работают без ошибок.

7. **Inventory/HUD**
   - Шаг: Открыть инвентарь после входа в мир.
   - Ожидаемый результат: Inventory/HUD корректно загружается без ретраев, ошибок и пустых данных после входа в мир.
