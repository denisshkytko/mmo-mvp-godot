# Аудит `assets/tiles` на пригодность для одноразмерных тайлсетов

Дата проверки: 2026-03-05.

## Что проверено
- Все PNG-файлы в `assets/tiles`.
- Размеры (ширина/высота) каждого файла.
- Кратность распространённым тайловым шагам (16/32/64).
- Однородность размеров внутри тематических папок.

## Общая статистика
- Всего PNG: **360**.
- Уникальных размеров: **97**.
- Самые частые размеры:
  - `32x32`: 84 файла.
  - `64x64`: 77 файлов.
  - `256x256`: 53 файла.
  - `128x128`: 28 файлов.
- Кратность размеров:
  - кратны 16: 277 файлов;
  - кратны 32: 251 файл;
  - кратны 64: 159 файлов;
  - **не кратны 16: 83 файла**.

## Где можно собрать «нормальный одноразмерный тайлсет»

### Готово почти «из коробки»
1. `assets/tiles/Forest/Land And Road` — полностью однородный набор `32x32` (64 файла).
2. `assets/tiles/Desert/Land And Road` — микс `64x64` и `256x256`, где `256x256` делится на блоки `64x64`.
3. `assets/tiles/Swamp/Land And Road` — аналогично: `64x64` + `256x256`.

### Можно использовать как объектные/декоративные слои
- `Forest/Decor`, `Swamp/Decor`, `Coast/Decor`, `Desert/Decor` содержат много корректных размеров (32/64/128), но также и нестандартные спрайты, поэтому это скорее набор декораций, чем чистый tile atlas.

## Где есть явные проблемы для единообразного тайлсета

Ниже папки, где размеры сильно разноформатные и/или не кратны 16:
- `assets/tiles/Forest/Grass` (`5x6`, `9x6`, `6x10` и т.д.).
- `assets/tiles/Forest/Stone` (`4x3`, `9x8`, `10x9` и т.д.).
- `assets/tiles/Forest/Camp` (разные нестандартные размеры, например `57x36`, `36x51`).
- `assets/tiles/Forest/Shadow` (размеры вроде `55x44`, `95x62`).
- `assets/tiles/Cities And Settlements/Decor` (много мелких нестандартных размеров, `23x21` и пр.).
- `assets/tiles/Cities And Settlements/House` (крупные спрайты разного размера: `154x149`, `156x135`, `116x112`).

Также в `Coast/Decor` есть крупные нестандартные тени (`191x148`, `169x74`, `89x47`), что нормально для декора, но плохо для сетки «один размер = один тайл».

## Вывод
Из текущего набора **можно** собрать нормальные одноразмерные тайлсеты, но **не из всех папок сразу**:
- для ground/road-слоёв — да (особенно Forest 32x32, Desert/Swamp 64x64);
- для decor/props — лучше использовать отдельным слоем объектов (Atlas/Scenes), а не как строгий единый TileSet.

## Практическая рекомендация по стандартизации
1. Выбрать базовый размер уровня (рекомендую `32x32` или `64x64`).
2. Разделить ассеты на 2 группы:
   - **Tile grid**: только кратные выбранному шагу;
   - **Objects/Decor**: всё остальное, размещать как сцены/декор.
3. Для `256x256` спрайтов в Land/Road нарезать на подтайлы выбранного размера.
4. Нестандартные ассеты (Grass/Stone/Shadow и часть Cities) либо пересобрать в atlas с паддингом до сетки, либо оставить исключительно как object layer.
