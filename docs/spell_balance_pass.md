# Spell balance pass (class-internal + cross-class)

## Что было сделано
Вместо ручного перебора сотен чисел в `.tres` добавлен единый runtime-нормализатор урона для всех основных damage-эффектов:

- `core/abilities/ability_balance.gd`
- подключен в:
  - `effect_damage.gd`
  - `effect_fireball.gd`
  - `effect_chain_damage.gd`
  - `effect_hailstorm.gd`
  - `effect_lucky_shot.gd`
  - `effect_multi_shot_damage.gd`
  - `effect_panjagan.gd`

## Принцип балансировки
Итоговый базовый урон способности умножается на фактор:

`total_factor = cast_factor * cooldown_factor * resource_factor * school_factor * class_factor`

с ограничением диапазона `[0.70 .. 1.45]`.

### Факторы
- `cast_factor`: быстрые каста (0-1с) штрафуются, длинные (3-4+с) усиливаются.
- `cooldown_factor`: способности с высоким КД получают бонус эффективности.
- `resource_factor`: дорогие по ресурсу способности получают небольшой бонус (до +10%).
- `school_factor`: легкая нормализация магии вниз (`0.94`), чтобы магический урон не обгонял физический «по умолчанию».
- `class_factor`: мягкая кросс-классовая нормализация (маг/жрец чуть вниз, воин/охотник чуть вверх).

## Ожидаемый эффект
- Быстрые спеллы перестают быть лучшими по efficiency.
- Долгие и/или с большим КД становятся ощутимо «вкуснее».
- Уравнивается эффективность магических и физических линий урона.
- Баланс внутри класса стабилизируется без ручной правки всех рангов.

## Почему такой подход
1. В проекте много способностей и рангов; ручной ретюн сейчас слишком дорог.
2. Единая формула даёт предсказуемость и быстрые итерации.
3. Поверх этого можно делать точечные правки outlier-спеллов уже после теста в бою.

## Следующий шаг (после полевого теста)
Снять метрики по 3 сценариям на каждом диапазоне уровней:
- слабый эквип: цель ~60/40 в пользу моба,
- средний эквип: цель ~70/30 в пользу игрока,
- хороший эквип: комфорт против 2-3 мобов, тяжело против толпы.

И затем докрутить только коэффициенты в `ability_balance.gd`, не трогая каждый `.tres`.
